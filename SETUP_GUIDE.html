1
Folder structure
Create a single folder for the entire project. All files must be in the same directory — the server serves index.html and register.html directly from this location.
vaultx/ ├── app.py ← Flask backend — download from this chat ├── index.html ← Main UI (login, dashboard, files) — download from this chat ├── register.html ← Public registration form — download from this chat ├── .env ← Your secret keys — you create this in step 5 ├── requirements.txt ← Python package list — you create this in step 3 ├── .gitignore ← Prevents secrets from being committed — you create this now └── vaultx.db ← SQLite database — created automatically on first run
Create .gitignore now — paste this content
.env
vaultx.db
venv/
__pycache__/
*.pyc
⚠ Do this before anything else. If you accidentally commit .env to a public repository, your Telegram bot token is exposed and must be regenerated immediately from @BotFather.
2
Python setup and virtual environment
A virtual environment isolates VaultX packages from everything else on your machine. This prevents version conflicts and keeps the project portable.
Open the VS Code terminal — press Ctrl + ` (backtick key, top-left of keyboard)
# Check Python version — you need 3.9 or higher
python --version

# If you see "python: command not found" try:
python3 --version
⚠ If Python is not installed, go to python.org, download the latest version for your OS, run the installer, then restart VS Code. On the Windows installer, check the box that says "Add Python to PATH".
Create and activate the virtual environment
# Step 1 — Create the virtual environment folder
python -m venv venv

# Step 2 — Activate it
#   Windows (Command Prompt or PowerShell):
venv\Scripts\activate

#   Mac or Linux:
source venv/bin/activate

# Step 3 — Confirm activation
# You should now see (venv) at the start of the terminal line, like this:
(venv) C:\Users\you\vaultx>
ℹ Every time you close VS Code and reopen it, you must run the activate command again before starting the server. The (venv) prefix disappears when the window closes.
3
Create requirements.txt and install packages
Four Python libraries power the entire backend: Flask (web server), requests (Telegram API calls), cryptography (key generation), and python-dotenv (.env file loading).
Create requirements.txt inside your vaultx folder — paste this exact content
flask
requests
cryptography
python-dotenv
Install all packages with one command
pip install -r requirements.txt
✓ You will see a long list of packages being downloaded and installed. Wait until you see "Successfully installed ..." before continuing. This takes about 30–60 seconds.
Verify installation
pip show flask
✓ If you see "Name: Flask" in the output, the installation worked. If you see an error, make sure (venv) is active in your terminal and run pip install -r requirements.txt again.
4
Set up your Telegram bot
VaultX uses one Telegram bot to store encrypted files. Each organisation gets its own private channel. The bot uploads and downloads files to the correct channel for each organisation automatically.
Create the bot — follow these steps inside Telegram
1
Open Telegram on your phone or desktop and search for @BotFather in the search bar. Open the official BotFather chat (it has a blue checkmark).
2
Send the message /newbot to BotFather. It will ask you to choose a name for your bot (this is the display name, e.g. "VaultX Storage Bot") and then a username ending in "bot" (e.g. "vaultx_storage_bot").
3
BotFather will give you a bot token that looks like this: 123456789:ABCDefGhIJKlmNoPQRsTUVwxyz — copy this token and save it somewhere safe. You will paste it into your .env file.
4
Now create a private channel for your first organisation. Tap the pencil/compose icon in Telegram → New Channel → give it a name (e.g. "Acme Corp Files") → set type to Private → tap Create.
5
Open the channel → tap the channel name at the top to open Channel Info → Administrators → Add Administrator → type your bot's username → select it → give it all permissions (especially "Post Messages" and "Delete Messages") → tap Save.
6
Type any message inside the channel (e.g. just "hello") — this is required so Telegram registers the channel in the API response.
7
Open this URL in your browser — replace YOUR_BOT_TOKEN with your actual token:
Get the channel ID from this URL
https://api.telegram.org/botYOUR_BOT_TOKEN/getUpdates

# In the JSON response that appears, look for a section like this:
# "chat": { "id": -1001234567890, "title": "Acme Corp Files", "type": "channel" }
#
# The negative number (-1001234567890) is your CHAT_ID for this channel.
# Copy it — you will enter it in VaultX when creating this organisation.
ℹ Repeat steps 4–7 for every organisation you plan to create. Each organisation must have its own dedicated private channel with the bot added as admin. If you try to use the same channel for two organisations, files will be mixed together.
⚠ If the URL returns an empty result like {"ok":true,"result":[]}, you forgot to send a message in the channel. Go send any message in the channel, then refresh the URL.
5
Create your .env file
The .env file stores your secret configuration. Python loads it automatically at startup. Never share this file or commit it to any repository.
Create a file named exactly .env in your vaultx folder — paste this and fill in your values
VAULTX_BOT_TOKEN=paste_your_full_bot_token_here
VAULTX_SECRET=type_any_random_string_of_32_or_more_characters
ℹ VAULTX_SECRET is used to sign session cookies. It can be any random string — the longer the better. Example: "kf82hd9sAp3nXq7rBwL4mYvZ0cTe1gUj". You only set this once and never change it (changing it logs everyone out).
⚠ On Windows, some editors add a .txt extension to .env and save it as ".env.txt". Check the file name in VS Code's file explorer. If it says ".env.txt", rename it to ".env" (no extension).
6
Add dotenv loading to app.py
Two lines at the top of app.py tell Python to read your .env file automatically every time the server starts. Without these lines, your bot token and secret key will not load.
Open app.py in VS Code and check the top of the file — these 2 lines must be present
from dotenv import load_dotenv
load_dotenv()   # reads .env automatically — must be before any os.environ.get() calls
✓ If you downloaded app.py from this chat, these lines are already there. Just double check by opening app.py and looking at the first 10 lines.
7
Start the VaultX server
Run the server and open the app in your browser. Keep the terminal open while you use VaultX — closing it stops the server.
Make sure (venv) is active, then run
python app.py
✓ You should see "VaultX → http://localhost:5000" printed in the terminal. The server is now running and waiting for requests.
⚠ Keep this terminal open. Do not close it or press Ctrl+C while using VaultX. To run other commands (like the bootstrap in the next step), open a second terminal tab with Ctrl+Shift+` or the + button in the terminal panel.
Open these URLs in your browser
http://localhost:5000            # Main app — shows the VaultX login screen
http://localhost:5000/register  # Public registration page for new organisations
8
Create the master admin account
This is your personal account — the provider/owner with full access to all organisations. This command runs once only. After that the endpoint is permanently locked.
Open a second terminal tab and run this command
# Mac or Linux:
curl -X POST http://localhost:5000/api/bootstrap \
  -H "Content-Type: application/json" \
  -d '{
    "username":     "masteradmin",
    "password":     "choose-a-very-strong-password",
    "display_name": "Master Admin"
  }'
✓ A successful response looks like: {"ok": true} — now go to localhost:5000 and log in with the username and password you just set.
Windows — PowerShell (if curl does not work)
Invoke-RestMethod `
  -Uri http://localhost:5000/api/bootstrap `
  -Method POST `
  -ContentType "application/json" `
  -Body '{"username":"masteradmin","password":"strong-password","display_name":"Master Admin"}'
ℹ Alternative: Install the VS Code extension "REST Client" by Huachao Mao. Create a file named test.http, paste the curl command above (without the backslashes), and click "Send Request" that appears above it.
What to do if you forgot your master admin password
# Stop the server with Ctrl+C
# Delete the database file
del vaultx.db       # Windows
rm vaultx.db        # Mac or Linux
# Restart the server and run the bootstrap command again
# Warning: this erases ALL users, organisations, and file metadata
✕ The actual encrypted files in Telegram are not deleted, but VaultX loses all the metadata (which file belongs to which org, folder structure, version history). Only delete vaultx.db as a last resort.
9
Create organisations
After logging in as master admin, go to the Organisations tab to create and manage organisations. Each organisation is completely isolated — users in one org cannot see anything from another org.
Fields required when creating a new organisation
Organisation name   →  Acme Corp
                         The name shown everywhere in the UI

Admin username      →  acme_admin
                         The login username for the organisation admin

Admin password      →  strong-password-here
                         You create this and share it securely with the org

Admin display name  →  Acme Corp Admin
                         The name shown in the dashboard header

Telegram channel ID →  -1001234567890
                         The channel ID from step 4 for this specific org

Storage limit (GB)  →  500
                         Maximum total storage for this org (warnings at 70%/90%/100%)

Plan                →  standard
                         starter / standard / enterprise — for your records only

Team passphrase     →  a-secret-encryption-key-for-this-org-only
                         Used by the browser to encrypt files before upload.
                         Every user in this org must use this exact passphrase.
                         Share it securely — not over email or chat.

Contact name/email  →  optional
                         For your reference — the person you sold this plan to
ℹ When you click Create, VaultX sends a test message to the Telegram channel to verify the bot can reach it. If verification fails, the organisation is not saved. Check that you added the bot as admin to that specific channel (step 4, step 5).
After creating an organisation — what to share with them
1
Share the login URL: http://your-server-address:5000
2
Share the admin username and password you set — use a password manager like 1Password or Bitwarden, not email or chat
3
Share the team passphrase — this is the AES-256 encryption key. Every user in the org needs it to upload and download files. Share it in person or via a secure password manager — if anyone outside the org gets this, they can decrypt the files.

11
Impersonation — entering an organisation's workspace
As master admin, you can enter any organisation's workspace in two modes. Both modes give you full master admin powers inside the org. The org admin is always notified when you enter.
Switch mode
# You see the org's full UI
# You can act as their admin
# Full master admin powers
# Blue banner shows at all times:
#  "Switch Mode — Acme Corp"
# One-click Exit button visible
ℹ Use Switch mode when you need to actively manage an org — create folders, move files, fix permissions.
Shadow mode
# You observe what the org sees
# You can still take actions
# Full master admin powers
# Orange banner shows:
#  "Shadow Mode — Acme Corp"
# One-click Exit button visible
ℹ Use Shadow mode when investigating a support issue — you can see exactly what the org sees without disrupting their work.
What gets logged when you impersonate
✓
Every action you take inside the org is logged as "master_admin acting as [org name]" — separately tagged from normal org activity
✓
Your entry and exit timestamps are recorded in both your master log and the org's own activity log
✓
The org admin receives an in-app notification the moment you enter: "Master admin (your name) has entered your workspace in Switch/Shadow mode"
✓
The org admin sees a read-only audit entry in their activity log (not real-time — appears after you exit)
12
Storage warning system
VaultX automatically monitors storage usage for each organisation and sends notifications at three thresholds. At 100% uploads are blocked until you approve a storage upgrade.
Three automatic thresholds
70
Warning (70%) — A yellow warning notification is sent to the org admin inside VaultX. Uploads still work normally. The org admin sees a banner on their dashboard showing current usage.
90
Critical (90%) — A red critical alert is sent to the org admin. Uploads still work but with a prominent warning. The org admin is advised to request a storage upgrade immediately.
100
Blocked (100%) — All uploads are immediately blocked. An automatic storage upgrade request is sent to you (master admin) on behalf of the org. You see a notification in your bell. The org admin also gets notified and can see the request status in their dashboard.
ℹ To approve a storage request: go to Storage Requests tab → click Approve → enter the new limit in GB (e.g. 1000 for 1 TB) → confirm. The org's storage warning resets and uploads resume immediately.

Common errors and exactly how to fix them
Every error below has been seen during setup. Follow the fix steps in order.
ModuleNotFoundError: No module named 'flask'
# Cause: the virtual environment is not active
# Fix: run the activate command in your terminal
venv\Scripts\activate      # Windows
source venv/bin/activate   # Mac or Linux
# Then try again:
python app.py
Telegram error: "chat not found" or "bot is not a member of the chat"
# Cause: the bot has not been added as admin to that org's channel
# Fix:
# 1. Open the Telegram channel
# 2. Tap the channel name at the top → Administrators
# 3. Add Administrator → search for your bot username → select it
# 4. Enable all permissions → Save
# 5. Try creating the org again in VaultX
Environment variable VAULTX_BOT_TOKEN is empty or None
# Cause 1: .env file is missing or has wrong name (.env.txt instead of .env)
# Fix: rename the file in VS Code file explorer to exactly ".env"

# Cause 2: load_dotenv() is missing from app.py
# Fix: add these 2 lines at the top of app.py:
from dotenv import load_dotenv
load_dotenv()

# Cause 3: no value set in .env
# Fix: open .env and make sure it looks exactly like this:
VAULTX_BOT_TOKEN=123456789:ABCDefGhIJKlmNoPQRsTUVwxyz
OSError: [Errno 98] Address already in use (port 5000)
# Cause: another process is using port 5000
# Fix option 1: find and stop the other process
lsof -i :5000       # Mac or Linux — find the process ID
kill -9 [PID]       # replace [PID] with the number shown

# Fix option 2: change VaultX to a different port
# Open app.py, scroll to the very last line, change 5000 to 5001:
app.run(debug=True, port=5001)
# Then open http://localhost:5001 in your browser instead
Bootstrap returns: "already initialized"
# Cause: a master admin account already exists in vaultx.db
# Fix: just log in at localhost:5000 with the credentials you already set
#
# If you genuinely forgot the password and need to reset:
# 1. Stop the server (Ctrl+C in the terminal)
# 2. Delete the database:
del vaultx.db      # Windows
rm vaultx.db       # Mac or Linux
# 3. Start the server again: python app.py
# 4. Run the bootstrap command again
✕ Deleting vaultx.db erases all users, all organisations, all folder structure, and all version history. The encrypted files in Telegram survive but cannot be linked to anything. Only do this if you are starting fresh from scratch.
Files upload successfully but download fails or gives wrong content
# Cause: wrong team passphrase entered during download
# The passphrase used to download must exactly match the one used to upload
# Fix: make sure every user in the org is using the same passphrase
# The passphrase is case-sensitive and space-sensitive
getUpdates URL returns empty result []
# Cause: no message has been sent in the Telegram channel yet
# Fix: open the channel in Telegram, type any message (e.g. "test"),
# then refresh the getUpdates URL in your browser
